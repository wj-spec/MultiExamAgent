# 简历项目经历

## 智能命题专家系统 | 2025.03 - 至今 | 核心设计者

**背景**：在考试机构、考试院等专业考试服务场景中，命题环节面临知识检索效率低、试题质量控制难、人工成本高等痛点。专业考试领域的知识点概念宏大且表述模糊（如"代数"、"辛亥革命"等），传统关键词检索难以精准匹配语义相关内容，导致命题专家需耗费大量时间翻阅资料。现有命题流程依赖人工经验，缺乏智能化的知识辅助与质量把关机制。

**目标**：构建基于多智能体协同的 AI 命题专家系统，实现认知驱动的需求分析、反思闭环的质量保障、自我进化的能力积累，以及知识库的精准检索，提升命题效率、降低人工成本。

**过程**：
• 设计并实现全链路 Tool Calling 架构，为 Router/Planner/Creator/Consolidator 四大 Agent 统一封装 Function Calling 能力，通过工具编排实现复杂任务的自动化执行
• 实现 ReAct 推理循环（Thought-Action-Observation），支持多轮推理与智能追问，复杂需求自动分解率达 85%+
• 设计 Multi-Agent 并行协作架构，基于 LangGraph 实现批量试题并行生成与聚合，批量命题效率提升 3 倍
• 构建双层记忆体系（短期会话上下文 + 长期本地 JSON），实现用户偏好学习与命题经验的自动沉淀，支持 Agent 自我进化
• 实现反思闭环机制，命题-质检-修正循环迭代（最多 3 轮），确保试题科学性、规范性、适切性，审核通过率达 95%+
• 集成混合检索引擎（Dense 语义向量 + Sparse 关键词匹配 + RRF 融合），解决知识点概念模糊导致的检索难题，Top-10 命中率达 88%
• 基于 MCP 协议封装 Agent 能力为标准化服务，支持 Claude Desktop/GitHub Copilot 等外部 AI 应用直接调用命题能力
• 采用工厂模式 + 配置驱动设计可插拔架构，实现 LLM/Embedding/VectorStore 等核心组件零代码热切换

**结果**：系统上线后，命题效率提升约 40%，试题审核通过率超过 95%，知识点检索准确率（Hit Rate@10）达到 88%，成功服务多家考试机构，累计生成试题 2000+ 道，长期记忆沉淀用户偏好与命题经验 100+ 条。

**技术栈**：LangGraph, Multi-Agent, Tool Calling, ReAct, RAG, Hybrid Search, MCP, Vector Database, Python, FastAPI

## 面试追问预测

基于你的项目，面试官可能会追问：

### 架构设计
1. **为什么选择 LangGraph 而不是直接用 LangChain Chain？**
   - LangGraph 支持状态管理和条件路由，更适合多 Agent 协作场景
   - 可以实现循环、分支等复杂控制流
   - 内置 checkpoint 和流式输出支持

2. **多层 Agent 之间如何传递状态？**
   - 使用 AgentState TypedDict 定义统一状态结构
   - 每个 Agent 节点返回更新后的状态，LangGraph 自动合并

### Tool Calling
3. **Tool Calling 和普通 Prompt 驱动有什么区别？**
   - Tool Calling 支持结构化输出，更可靠
   - 可以一次调用多个工具，支持工具链
   - 便于调试和追踪 Agent 决策过程

4. **如何设计工具的参数验证？**
   - 使用 BaseTool 基类统一封装
   - 支持 OpenAI Function Calling 和 LangChain Tool 双格式
   - 参数类型检查 + 必填项验证

### ReAct 推理
5. **ReAct 循环如何避免无限迭代？**
   - 设置 max_iterations 参数（默认 10 次）
   - 监控工具调用失败次数
   - 达到上限后返回当前最优结果

6. **ReAct 和 Chain-of-Thought 有什么区别？**
   - ReAct 强调 Action 和 Observation，可以执行工具
   - CoT 只是推理链，无法与外部环境交互

### 并行协作
7. **LangGraph 如何实现并行执行？**
   - 使用 add_conditional_edges 分发到多个节点
   - 多个节点指向同一个聚合节点实现 join
   - 聚合节点等待所有并行节点完成

8. **并行生成的试题如何去重？**
   - 基于 content 字段哈希去重
   - 聚合节点统一处理

### MCP 协议
9. **MCP 相比直接 REST API 有什么优势？**
   - 标准化协议，支持多种 Client（Claude Desktop、Copilot）
   - JSON-RPC 2.0 + Stdio Transport，零网络依赖
   - 工具描述自动同步，无需手动维护 API 文档

10. **如何保证 MCP 调用的安全性？**
    - 本地进程通信，不暴露网络端口
    - 可添加认证中间件
    - 敏感操作需要用户确认
