## 二、参考答案

### <a id="a-存储架构"></a>Q: 系统里有哪几类存储？

**参考答案**：
系统采用多存储架构，各存储职责明确：

| 存储 | 位置 | 职责 |
|------|------|------|
| **Chroma 向量库** | `data/db/chroma/` | Dense 向量 + metadata，语义检索 |
| **BM25 倒排索引** | `data/db/bm25/` | 稀疏索引，关键词检索 |
| **ingestion_history.db** | `data/db/` | SHA256 文件处理记录，防重复摄取 |
| **image_index.db** | `data/db/` | image_id → 文件路径映射 |
| **long_term_memory.json** | `data/memory/` | 长期记忆，用户偏好与命题经验 |
| **会话内存** | 运行时 | 短期上下文，Agent 对话状态 |

**为什么不能只用一个**：
- 向量库支持语义检索但无法做精确关键词匹配
- BM25 补足关键词检索能力，两者结合才是 Hybrid Search
- ingestion_history.db 实现文件级幂等去重
- JSON 存储实现长期记忆的持久化与可读性
- 内存存储满足 LLM 上下文窗口需求

### <a id="a-hit-rate"></a>Q: Hit Rate@K 是怎么计算的？

**参考答案**：

$$HitRate@K = \frac{\text{Top-K 结果中至少命中一条 Golden Answer 的查询数}}{\text{总查询数}}$$

对 Golden Test Set 中每条 `(query, expected_chunks)`，取 Top-K 检索结果，至少一条匹配则 hit=1，否则 hit=0。Hit Rate@K = 命中次数 / 总 case 数。

**@K 含义**：只要正确文档出现在 Top-K 内即算命中，不要求排第一。@10 = 正确文档在 Top-10 内即可。

**测量要点**：
- Golden Test Set 需人工标注或高质量自动生成
- 每条 case 包含 `(query, expected_chunk_ids)`
- 报告应说明测试集规模、标注方式、置信区间

### <a id="a-多租户"></a>Q: 多租户扩展需要改动什么？

**参考答案**：

**核心改动点**：
1. **用户管理系统**：认证/鉴权层（JWT/OAuth）、用户表、权限模型
2. **数据隔离**：
   - Chroma：按 collection 或 metadata.tenant_id 隔离
   - BM25：索引分片或前缀隔离
   - 长期记忆：JSON 文件按租户拆分或引入数据库
3. **MCP Server**：从 Stdio Transport 切换到 HTTP+SSE，支持远程多连接
4. **配置层**：租户级配置覆盖（不同租户可用不同 LLM/Embedding）

**最难点**：
- 向量库跨租户检索的权限边界
- 租户间资源配额与隔离
- 现有 Stdio Transport 不支持多 Client 并发，必须重构传输层

---

### <a id="a-mcp协议"></a>Q: MCP 是什么规范？传输协议是什么？

**参考答案**：
MCP（Model Context Protocol）是 **Anthropic** 提出的开放协议，基于 JSON-RPC 2.0，定义 AI Client 与外部工具/数据源之间的标准通信接口。任何合规 Client（Copilot、Claude Desktop）即插即用。

本项目采用 **Stdio Transport**：Client 以子进程启动 Server，stdin/stdout 通信，日志走 stderr，零网络依赖。

**为什么选 Stdio 而非 REST API**：
- 零网络依赖，无需端口/鉴权配置
- 天然适合本地私有知识库场景
- stdout 只输出合法 MCP 消息，stderr 走日志，职责清晰
- 局限：不支持远程调用、不支持多 Client 并发

### <a id="a-可插拔架构"></a>Q: 新增一个 Embedding Provider 需要改哪些文件？

**参考答案**：
需要改 **3 处**，遵循开闭原则：

1. **新建** `src/libs/embedding/your_provider.py`：继承 `BaseEmbedding`，实现 `embed_texts()` 等接口方法
2. **修改** `src/libs/embedding/factory.py`：在 `provider_map` 中注册 `"your_provider": YourProviderClass`
3. **修改** `config/settings.yaml`：将 `embedding.provider` 改为 `"your_provider"`

**候选人回答问题**：说"仅仅修改配置文件"是不完整的，漏掉了：
- 新建 Provider 实现类
- 在 Factory 中注册映射关系

这说明对可插拔架构的实现细节理解不够深入。
